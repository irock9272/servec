Architecture

How servec works

Design Principles

Minimal Dependencies

Written in pure C with only standard library calls. Single compiled binary with no external dependencies to manage.

Thread Pool Architecture

Pre-allocated worker threads handle incoming requests. Eliminates per-request thread creation overhead. Configurable pool size.

Direct File Serving

Files served directly with minimal buffering. Automatic MIME type detection based on file extensions.

Security Features

Directory Traversal Protection

Detects and blocks .. sequences in request paths. Prevents access to parent directories.

Sensitive Path Blocking

Hardcoded blocks on .git/, .env, src/, build/, and Makefiles prevent accidental exposure.

Extension Whitelisting

Optional configuration to restrict served file types. Default allows common web assets (html, css, js, images, text).

Configuration

servec reads from ~/.config/servec/config with these options:

port

Listen port. Default: 6969. Can be overridden via command line argument.

root_dir

Directory to serve files from. Default: current directory.

error_404 / error_500

Custom HTML files for error responses. Falls back to simple defaults if not configured.

thread_pool_size

Number of worker threads. Default: 4. Adjust based on system resources and expected load.

Request Handling

1

Accept Connection

Main thread accepts incoming TCP connection

2

Submit to Pool

Connection handed to thread pool for processing

3

Parse Request

Worker thread reads HTTP request and validates path

4

Security Check

Checks for traversal attacks, extension whitelist, and blocked paths

5

Serve Response

Files served with appropriate MIME type, or error pages returned
