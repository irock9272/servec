Core Features

What makes servec the future of web serving

⚡

Lightning Fast

Written in pure C for maximum performance. No unnecessary overhead, just raw speed.

Response Time: <1ms Memory Usage: Minimal
🛡️

Security First

Built-in protection against directory traversal attacks and sensitive file access.

Blocked: .git, .env, src/, build/ Extension Whitelisting
🎨

Custom Error Pages

Fully customizable 404 and 500 error pages. Serve exactly what your users expect.

404: Page Not Found 500: Server Error
🔧

Configuration

Flexible configuration system with support for custom ports, root directories, and more.

Config: ~/.config/servec/config Format: INI-style
📁

MIME Type Detection

Automatic content-type detection for HTML, CSS, JavaScript, images, and more.

HTML, CSS, JS PNG, JPG, GIF
🔄

Thread Pool

Multi-threaded request handling with configurable worker pool for high concurrency.

Default: 4 threads Configurable

Configuration Example

# servec configuration file
port = 6969
root_dir = ./public
error_404 = ./templates/error_404.html
error_500 = ./templates/error_500.html

Why servec?

Compiled Binary

Written in C and compiled to a single executable. No runtime dependencies or interpreters.

Thread Pool

Built-in multi-threading with configurable worker pool for handling concurrent requests efficiently.

Security by Default

Directory traversal protection and configurable file whitelisting prevent common vulnerabilities.
